# Decisions

This folder records significant design and product decisions made during development.

Each file should cover one decision: what was decided, why, and any alternatives that were considered.

## File naming

`YYYY-MM-DD-short-description.md`

## Template

```markdown
# Title

**Date:** YYYY-MM-DD  
**Status:** decided | superseded | revisited

## Decision

What was decided.

## Reason

Why this option was chosen.

## Alternatives considered

What else was evaluated and why it was ruled out.
```
